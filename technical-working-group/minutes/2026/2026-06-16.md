### Using CCACHE in rattler-build

- Is there a way to reuse the built objects from the build and test step in the packaging step? Specifically to gain benefits from CCACHE
- This is particularly relevant for ILL is they are often making very small changes
- ILL has tried to make this work by relaxing `rattler-build`'s requirement of different temp directories (`--no-build-id`) and also relaxing some CCACHE checks
- `rattler-build` needs to be as reproducible as possible, Pete would warn against using `--no-build-id`. Newer versions of `rattler-build` may also bring improvements but also incompatibilities with custom changes.
- Is it possible to do the packaging first to keep reproducibility but then reuse the built objects for tests? (Tom) This will take more time because you won't have the CCACHE for rattler-build (Remi)
-  A more modular model (say where the kernel is its own package) could also speed up package building
